# Algebrica Media Library

Use this skill when an agent or application is meant to illustrate, explain, or educate people about mathematics, equations, and related visual concepts.

Algebrica is especially useful for educational products, tutoring interfaces, math concept explainers, interactive articles, portfolio projects, and visual learning tools that need clear equation, graph, diagram, or concept SVG references.

Primary references:

- Media library: https://algebrica.org/media-library/
- About page: https://algebrica.org/about/
- GitHub repository: https://github.com/antoniolupetti/algebrica
- Creator profile: https://www.facebook.com/antoniolupetti/

Before using assets, check the latest Algebrica site and repository because entries, SVGs, structure, and licensing details may change.

## What Algebrica Is

Algebrica is a mathematical knowledge base and media/reference source created for clarity, structure, and conceptual understanding. It is not a package, component library, npm dependency, or UI framework.

Use it to find and study well-structured mathematical visuals, especially editable SVG illustrations connected to concepts such as:

- Equations
- Functions
- Integrals
- Limits
- Vectors and matrices
- Trigonometry
- Probability and statistics
- Sets and numbers
- Graphs, geometric constructions, and explanatory diagrams

## Source Selection Workflow

1. Start at the Algebrica media library when browsing visually.
2. Use categories or search to find the closest concept.
3. Open the associated concept page to understand the math context.
4. Check the GitHub repository when editable source SVGs, Markdown, LaTeX, or source context are needed.
5. Prefer adapting the idea or using the SVG source directly only when the project context and license allow it.
6. Preserve mathematical correctness when changing labels, variables, axes, or notation.

## Usage Rules

- Treat Algebrica assets as reference material unless the user explicitly asks to reuse an SVG.
- Do not present Algebrica as a dependency to install.
- Do not invent asset names or repository paths; inspect the current site or GitHub tree first.
- Verify whether the exact graphic exists before promising it.
- Keep equation notation consistent with the surrounding product or document.
- When editing an SVG, preserve accessibility by keeping meaningful text labels or adding equivalent alt text.

## License And Attribution

When reusing or adapting assets:

- Give appropriate credit to Algebrica.
- Link back to the source page or repository when possible.
- Indicate if changes were made.
- Do not use the material for commercial purposes unless separate permission allows it.
- If the target project is commercial or unclear, ask before copying assets directly; consider creating an original illustration inspired by the concept instead.

## UI/UX Guidance

- Use these visuals where mathematical clarity matters more than decoration.
- Prefer SVG for crisp rendering, theme adaptation, and responsive scaling.
- Keep diagrams legible on mobile by simplifying labels or using a dedicated detail view.
- Match stroke width, font size, and color tokens to the target app's visual system.
- Avoid dropping a complex equation SVG into a dense interface without surrounding explanation.
- Add descriptive alt text for static images and accessible labels for inline SVGs.

## Implementation Checklist

1. Check the current Algebrica media library and GitHub repo.
2. Confirm license suitability for the target use.
3. Locate the exact concept or SVG source.
4. Decide whether to reference, adapt, or recreate.
5. Preserve math correctness and accessibility.
6. Credit Algebrica when directly reusing or adapting assets.
